weixin_miniprogram-1

探索小程序，技术框架层面

为什么探索？

  • 最近朋友做蛋糕生意的，提议如果能做个小程序辅助销售就好了，而且这不是少数的需求。正好自己也知道，于是摸索学习一下。

环境准备

  1. 前往微信公众平台申请小程序，一个邮箱只能申请一个
  2. 保存好app id等信息
  3. 下载微信小程序开发IDE
  4. 创建工程

目录接口

  • 一个页面page = js+json+wxml+wxss目录结构
    • 该目录结构对应着前端开发三大模块，javascript=js，html=wxml，wxss=css，但是在这个基础上微信做了改造，写法上很相似。但增加了一些自己的方法，如wx:for,wx:if,wx:else等等,官方做的很好，文档写的很详细
  • js
    • 自己对js不太很了解，但小程序这个很容易懂。js里面对应着一个页面的生命周期，如onLoad,onHide等生命周期的函数，还有data:等数据函数。
    • 自己也可以增加自定义函数，绑定事件，或者增加网络请求等
    • 有个坑：data方法和onLoad都会进行数据初始化，小心覆盖数据。
  • json
    • 我没用到
  • wxml
    • 这里面是标签语言，具体参看官方文档是最好的
    • 几个常用的标签,主要以做分块，其他还有,,还有官方提供的可以做逻辑判断或者for循环
  • wxss
    • 这里面跟css很像，自己也不是很明白，写页面挺简单的，猜一猜就能写
    • 学习了flex布局，参看 文档

实现细节

  • UI方面
    *
  • 网络方面
    1. 官方提供了wx.request,很方便
    2. 自己做了post和get封装，包一下就好，简单易用
    3. 调试方法：利用ide提供的sources可以端点调试，console打印数据
    4. 遇到的坑：
      • 【url无法访问】问题是没有开启ssl认证，idle右上方点击，授权ssl认证。另一方面，后台需要配置request的host地址，不然也是不行的
      • 【接口返回数据错误】原因：注意微信小程序后台url和请求的url是否一致
    5. 解析数据：这一点前端正是很方便，会解析好，a.data.result这样直接取，取好之后通过page的setData方法，调用this.setData({xx:dataArr})就好了，外面函数通过this.getData获取
  • 数据存储
    • 页面自身，通过setdata方法操作
  • 页面传值
    • 通过url，url？param1=xxx,在新页面中，onLoad中options.param1可以直接取到
    • 通过app全局存
    • 通过wx.storage来做，这边是key-value形式
  • webview处理
    1. 新建一个page
    2. wxml中，需要声明一个脚本
    3. 创建webview
    4. 直接wx.navigateTo()跳转就好了
  • 其他
    • 首页改变，在app.json中，调整pages顺序即可
    • 可以在app.json中设置全局属性，如timeout

iOS - Calendar EventKit

iOS日历踩坑

实践起因

  • 产品最近想给app添加日历提醒功能，弥补推送不及时和到达率不高的问题

Calendar EventKit 介绍

  • 之前做过siriKit的开发，并结合产品功能添加到app中，EventKit较siriKit模块更简单一些。

  • EventKit分类

    • Reminder: 增加提醒时间，这类事件会增加到“提醒事项”app中
    • Event: 这是普通事件，比如Alarm闹铃

  • 具体详细的介绍，参看帖子

    • 传送门

EventKit 遇到的坑

  • 坑一： Error Domain=EKErrorDomain Code=14 “日历没有来源” UserInfo={NSLocalizedDescription=日历没有来源}
    • 这是个比较坑的地方，首先我们搞清楚日历来源是什么？在代码中是EKSource的值（EKSource <0x1746cdba0> {UUID = B6EE2BCC-3355-4C24-813A-52203FF6BF86; type = Local; title = Default; externalID = (null)}） 这样一串数据
    • 日历来源一般选择local
    • 导致出现问题的原因有两个：
      • 其一，是否提前让用户授权许可，注意这一点一定要在EKEventStore创建前（尽量放在app主页面viewDidLoad时候）。我吃了这个亏，在EKEventStore创建后再调用授权，这样就导致EKEventStore对象中没有sources
      • 其二，calendar对象source对象一定要赋值，否则会返回创建日历提醒成功，但实际上并没有事件存在
  • 坑二：在删除多个事件的时候，记得使用事务提交。否则只会删一条而出错
    • [self.mEventStore removeReminder:reminder commit:NO error:&err]，commit使用no，意思是不是不提交。数组遍历调用完成后，再[self.mEventStore commit:&err]

总结

  • 这个功能苹果提供的api并不算友好，比如没有赋值calendar的resource时候，仍然能创建成功；比如创建calendar的错误提示信息过少。EventKit是个比较有用的功能，特别在工具app中，可以弥补推送的不足。踩完这些坑，神清气爽！

PS

  • 晚点整理一份demo提交到github。

FFMPEG(一)

FFMPEG 是什么？

  • A complete, cross-platform solution to record, convert and stream audio and video.
  • FF的意思是Fast-forward
  • MPEG全称是moving picture expert group
  • 也就是一种更高效的，移动图像解决方案

MPEG的发展历史

  • 具体可参考 link
  • 几个重要的版本
    • mpeg1：第一代版本，无论从图像分辨率还是音频采样率上来讲，都比较一般，代表产品是VCD。这也奠定了我们今后音视频的标准。比如音频：32khz(广播效果),44.1khz(mp3效果),48khz(高清效果)。
    • mpeg2：mpeg2在mpeg1基础上做了升级，视频分辨率和音频采样率都更丰富，代表产品是DVD。
    • mpeg4：4代定下了h.264编码规范，这在今后的视频应用中都是很重要的。
    • mpeg7：7代中推出了pull和push的概念，即我们想给视频素材增加标签，则push相应的信息上去；想要搜索符合”周杰伦唱片”的话，就pull下来。这对作品的分类和知识产权的保护都是很重要的。

FFMPEG应用案例 - ijkplayer

  • 介绍：这是bilibili的开源视频播放器，应用广泛
  • 获取方式：
    * https://github.com/Bilibili/ijkplayer
* 编译工程：
* 1. sh ./init-ios.sh
* 2. sh ./init-ios-openssl.sh
* 3. ./compile-ffmpeg.sh clean &  ./compile-ffmpeg.sh clean  all

FFMPEG目录结构及功能

  • libavcodec:用于各种类型声音/图像编解码；
  • libavutil：包含一些公共的工具函数；
  • libavformat：包含多种多媒体容器格式的封装、解封装工具
  • libswscale：用于视频场景比例缩放、色彩映射转换；
  • libpostproc：用于后期效果处理；
  • libavdevice：用于音视频数据采集和渲染等功能的设备相关
  • libswresample：用于音频重采样和格式转换等功能
  • libavfilter：包含多媒体处理常用的滤镜功能

应用举例

查看视频基本信息

  • ffmpeg -i test.mp4
  • ffmpeg -i test.mp4 -vcodec libx264 -preset fast -crf 32 -y -vf “scale=1280:720” -acodec libmp3lame -ab 32k out-compress.mp4
  • -crf: 这是用于指定输出视频的质量，取值范围是0-51，默认值为23，数字越小输出视频的质量越高。这个选项会直接影响到输出视频的码率。一般来说，压制480p用20左右，压制720p用16-18

把音视频替换背景音乐：

  • ffmpeg -i test.mp4 -vcodec copy -an videoNoAudioSubtitle.mp4
  • ffmpeg -i videoNoAudioSubtitle.mp4 -i ts-music.mp3 -vcodec copy -acodec copy output.mp4

制作一个类似抖音小视频

  • ffmpeg -y -i test.mp4 -ignore_loop 0 -i jump.gif -filter_complex overlay=x=0:0:shortest=1 test_out8.mp4
  • 压缩gif：ffmpeg -i jump.gif -vf scale=iw/4:-1 resize_jump.gif
  • 格式转换：ffmpeg -y -i test.mp4 -ignore_loop 0 -i resize_jump.gif -filter_complex overlay=x=50:30:shortest=1 test_out8.mp4

视频流和视频

  • 视频流是基于编码的，常见的流格式是h.264
  • 常说的视频是视频+音频混制而成，比如.avi, .mp4等等都是。

常见的流媒体协议

  • RTSP+RTP
  • RTMP
  • RTMFP
  • MMS
  • HTTP
